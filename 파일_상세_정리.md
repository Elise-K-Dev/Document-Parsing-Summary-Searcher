## 3. 왜 제외했는가

### zip

- 현재 `zip`은 문서 본문보다 백업본, 설치묶음, 원본 보관용 성격이 강하다.
- 그대로 파싱 대상으로 넣으면 압축 해제 정책, 내부 파일 선택 기준, 중복 문제를 같이 풀어야 한다.
- 그래서 현재는 `binary_or_media`로 유지했다.

### 잠금파일과 임시파일

- `~$*`, `.tmp`는 사용 중 파일 흔적이거나 임시 산출물이다.
- 문서 본문이 아니거나 불완전한 경우가 많아서 제외했다.

### 바로가기

- `.lnk`는 실제 문서가 아니라 참조 포인터다.
- 검색 대상에 넣으면 중복·경로 깨짐·원본 불명확 문제가 생긴다.

### 랜섬 흔적

- `README.html`은 랜섬노트다.
- `.yvfoelvl`은 암호화된 파일이라 정상 문서가 아니다.
- 이들은 검색 대상에서 제외하는 수준을 넘어, 원본 트리 밖 `data/quarantine/ransom`으로 격리했다.

### 미지원 확장자

- 이미지, 동영상, 설치 파일, 특수 확장자는 현재 검색 파이프라인의 목표 범위 밖이다.
- 예: `.jpg`, `.heic`, `.mp4`, `.exe`, `.msi`

## 4. 왜 포함시켰는가

### xlsx, xlsm

- 실제 업무 이력과 계획, 점검, 부품, 날짜, 장비번호가 표 구조로 많이 들어 있다.
- 현재 검색 가치가 가장 높아서 초기부터 핵심 대상이었다.

### docx

- 주간/월간/업무 보고 계열 문서가 많다.
- 행 기반 근거 탐색은 어렵지만, 문단 단위 검색만으로도 보고서 본문 증거를 찾을 수 있다.

### pptx

- 교육자료, 발표자료, 보고자료에 텍스트가 존재한다.
- 슬라이드 단위로 파싱하면 검색 근거로 활용 가능하다.

### pptm

- 실제 데이터셋에 `.pptm` 파일이 있었고, 내부 구조는 `pptx` 계열 OOXML과 동일한 방식으로 읽을 수 있다.
- 매크로 실행이 목적이 아니라 텍스트 추출이 목적이므로, 파싱 범위에 편입했다.

현재 결과:
- `.pptm` 문서 2건 중 1건은 SHA256 기준 완전 중복이라 `duplicate_skip`
- 1건은 실제 적재 완료

## 5. 어떻게 수정했는가

### 전처리 규칙 수정

수정 파일:
- `code/scripts/preprocess_documents.ps1`
- `docs/전처리/전처리_제외규칙.txt`

수정 내용:
- 허용 확장자 목록에 `.pptm` 추가
- 따라서 `.pptm`이 `unsupported_extension`에서 빠지고 `parse_ready`로 승격됨

### 스테이징 수정

수정 파일:
- `code/scripts/rag_stage.rs`

수정 내용:
- `manifest.csv`의 `rag_path`를 절대경로로 기록하도록 변경
- 백엔드 실행 디렉토리와 무관하게 스테이징 파일을 찾도록 보강

### 백엔드 파서 수정

수정 파일:
- `code/evidence_backend/src/ooxml_ingest.rs`
- `code/evidence_backend/src/app.rs`
- `code/evidence_backend/src/db.rs`

수정 내용:
- `.pptm`을 `.pptx`와 같은 OOXML 파워포인트 파서로 처리
- ingest 대상 확장자 목록에 `.pptm` 추가
- 실제 적재 가능 문서 목록 조회에도 `.pptm` 반영

### 데이터 정리 수정

수정 내용:
- 랜섬 흔적 4건을 `data/quarantine/ransom`으로 이동
- 전처리 재실행
- 스테이징 재생성
- 매니페스트 재적재
- `.pptm` 문서 단건 적재 확인

## 7. 아직 포함하지 않은 것

- `zip`
- `pdf`
- `txt`
- `doc`
- `xls`
- `ppt`

설명:
- 전처리 허용과 실제 파싱 지원은 일부러 분리했다.
- 후보로 남겨두되, 파서가 준비되지 않은 포맷은 검색 인덱스에 무리하게 넣지 않는다.

## 8. 현재 남은 실제 손상 파일

- 문서 ID: `4307`
- 파일명: `HJIT 주간 업무 보고(7월 1주차)_정비팀_170626.docx`

## 9. 결론

현재 파일 정책은 “문서 검색 품질을 높이는 쪽으로 보수적으로 포함하고, 위험하거나 비문서적인 것은 명시적으로 제외 또는 격리한다”로 정리되어 있다. 최근 변경의 핵심은 `pptm` 편입과 랜섬 흔적 격리다.
